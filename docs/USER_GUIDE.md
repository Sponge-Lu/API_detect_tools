# API Hub Management Tools - 用户使用指南

## 简介

**API Hub Management Tools** 用于管理和运维多个 API 中转站，适合同时维护多站点、多账号、CLI 配置和代理路由的用户。

当前版本为 **v3.0.4**。相较 `v2.1.24`，最明显的变化是：

- 新增 `数据总览` 首页，并将站点数据与路由数据拆为子页
- 新增 `本地路由` 工作台与对应主进程路由代理服务
- 新增 `日志` 页，支持会话事件与路由请求尝试两类子页
- 支持账号级自动刷新
- 主界面重构为侧边栏工作台
- 主题系统收敛为 `Light` / `Dark`

## 界面结构

当前一级页面包括：

- `数据总览`
- `站点管理`
- `自定义 CLI`
- `站点检测`
- `LDC 积分`
- `本地路由`
- `日志`
- `设置`

说明：

- `数据总览` 包含 `站点数据` 与 `路由数据` 两个子页。
- `日志` 包含 `会话事件` 与 `路由日志` 两个子页。
- `模型重定向`、`CLI 可用性`、`代理统计` 不再作为一级页面展示；相关能力已整合进 `本地路由`、`站点检测` 与 `数据总览`。

### 视觉与主题

- 使用统一产品级原语：`AppButton`、`AppInput`、`AppModal`、`DataTable`、`AppCard`、`AppIcon`
- 主题仅保留两套预设：`Light`、`Dark`
- 默认窗口尺寸：`1400 x 900`
- 最小窗口尺寸：`1200 x 700`

## 核心功能

### 1. 站点与账号管理

- 在一个页面中维护多个 API 站点
- 支持站点分组、排序、搜索和缓存数据展示
- 支持一个站点绑定多个账号
- 支持按账号配置自动刷新开关和间隔
- 支持按浏览器 Profile 直接打开站点

### 2. 自动认证与登录

- 添加站点时可直接输入 URL，通过浏览器登录后自动捕获凭证
- 若登录后站点未返回 `access_token`，系统会尝试自动补建令牌
- 遇到 Cloudflare 或挑战页时，会回退到浏览器模式
- 登录浏览器与检测浏览器已分离，减少批量检测和登录流程冲突

### 3. 站点检测与统计

- 查看余额、今日消费、请求数、RPM、TPM、模型数量等指标
- 批量检测时支持逐站即时更新
- 支持站点级与账号级自动刷新
- 支持全局排序，排序状态会自动保存

### 4. API Key 与签到

- 可视化创建、删除、查看 API Key
- 支持配额、过期时间、分组等参数
- 自动检测签到能力
- 支持单站签到与批量签到
- 支持浏览器模式完成被挑战站点的签到

### 5. CLI 配置与兼容性

支持以下 CLI：

- Claude Code
- Codex
- Gemini CLI

当前行为说明：

- 站点页、统一 CLI 配置抽屉、自定义 CLI 配置编辑器和路由探测统一通过真实 CLI wrapper 执行测试
- 测试会使用临时目录隔离 `HOME` / `CODEX_HOME` / Gemini 的 `HOME/.gemini`，正常情况下不会改写本机真实 CLI 配置
- 手动 CLI 测试与路由自动探测结果会同步写入 `routing.cliProbe`，站点卡片、CLI 配置抽屉和 CLI 可用性矩阵都会读取最新结果
- **Codex**：只测试并生成 Responses API 配置，不再使用 Chat Completions 模式
- **Gemini CLI**：会阻断已知内部辅助模型请求，避免未选中的 helper/default 模型被静默路由并产生额外费用

你可以完成以下操作：

- 为每个 CLI 选择 API Key 和模型
- 查看配置预览
- 直接编辑配置内容
- 一键写入本地配置文件
- 选择合并模式或覆盖模式应用配置
- 管理独立于站点检测结果的“自定义 CLI 配置”

### 6. 数据总览

`数据总览` 是当前默认首页，分为两个子页：

#### 站点数据

- 查看站点余额、今日消费、请求量、Token 与模型数量
- 查看站点每日快照趋势
- 查看签到状态与站点资源榜单

#### 路由数据

- 查看路由请求量、成功率、Token 与首字响应 / 会话时间分位数
- 按全部、站点或自定义 CLI 过滤路由桶级数据
- 使用运行趋势、模型热力、通道散点和模型到通道 Sankey 分析路由健康
- 运行趋势支持 `24h` 与 `7d` 视窗；即使当前只有部分路由桶数据，也会显示完整小时 / 日期 X 轴，首个真实数据点之前的空桶只显示时间标签

### 7. 本地路由

`本地路由` 是本机统一代理与模型重定向配置工作区。

#### 模型重定向

- 配置统一服务端口、统一 API Key
- 创建或修改路由规则
- 维护模型注册表与覆盖项
- 配置各模型的站点优先级、API Key 优先级与运行时熔断策略

#### CLI 可用性

- CLI 可用性矩阵当前位于 `站点检测` 相关工作区
- 可查看 7 天探测历史，历史条按一次探测批次聚合
- Header 右侧提供定时探测开关、小时制间隔输入和立即探测按钮

#### 代理统计

- `本地路由` 页可查看代理运行状态、默认模型和服务配置
- 查看健康检查结果
- 更完整的聚合统计与分布数据位于 `数据总览 -> 路由数据`

### 8. 日志

- `会话事件` 展示当前 renderer 会话中的通知与关键操作
- `路由日志` 展示本地路由代理逐条请求尝试
- 路由日志以无卡片、带表头的横向滚动单行表格展示 CLI 图标、请求 / 重试、原始模型、路由目标、Token（总 / 输入 / 输出 / 缓存写 / 缓存读）、参考金额、用时 / 首字、纯数字状态码与时间；失败尝试会在第二行显示错误信息
- 直连配置来源会以 `直连配置 / 配置名` 展示；站点 / 账户 / 分组 / API Key 路径仅适用于普通站点来源

### 9. Linux Do Credit

- 读取 `credit.linux.do` 的余额、收入、支出、交易记录
- 查看最近统计数据
- 选择支持 LDC 支付的站点并发起充值跳转

### 10. 备份、恢复与更新

- 本地自动备份稳定配置
- 从历史备份恢复
- 使用 WebDAV 进行 manifest 配置包云端备份
- 应用内检查更新、下载和安装
- 配置损坏时会优先尝试从最近备份恢复
- 运行态缓存、路由状态、探测历史和分析桶已拆分到 bounded sidecar 文件，避免高频运行态写入污染 `config.json`

## 常用使用路径

### 添加站点

1. 打开 `站点管理`
2. 新建站点并填写 URL
3. 选择自动登录或手动输入凭证
4. 登录完成后等待系统抓取账号信息

### 配置账号级自动刷新

1. 打开站点详情或账户相关入口
2. 为目标账号设置 `auto_refresh`
3. 设置刷新间隔，最小为 15 分钟
4. 保存后等待定时器生效

### 生成 Codex 配置

1. 打开 `自定义 CLI` 或站点 CLI 配置入口
2. 选择 `Codex`
3. 选择 API Key 和模型
4. 预览生成结果
5. 点击应用

说明：当前生成的 Codex 配置固定使用 `Responses API`。

### 使用 Route 工作台

1. 在侧边栏进入 `本地路由`
2. 配置代理服务端口、统一 API Key、默认 CLI 模型与模型重定向规则
3. 在 `站点检测` 查看 CLI 可用性探测与历史
4. 在 `数据总览 -> 路由数据` 查看路由统计图表
5. 在 `日志 -> 路由日志` 查看逐条代理请求尝试

说明：稳定路由配置会写回 `config.routing`；运行态统计、路径暂停、探测历史和分析桶保存在应用数据目录的 sidecar 文件中。

## FAQ

### 为什么登录浏览器没有立刻关闭？

- 可能仍在等待你完成登录或通过挑战页
- 当前版本会把登录流程放在独立登录浏览器中，避免打断检测任务
- 若长时间无进展，请检查是否仍停留在验证页或登录页

### 为什么有些站点需要浏览器模式才能检测？

- 某些站点接口会被 Cloudflare、WAF 或登录态限制拦截
- 当前版本会统一识别挑战页并切换到浏览器上下文继续获取数据

### 为什么 Codex 不再测试 Chat Completions？

- 当前版本已经把 Codex 配置与测试收敛到 Responses API
- 这是 v3 主线相比 `v2.1.24` 的一个行为变化

### 为什么 CLI 测试失败后很快结束？

- CLI 测试会通过本地路由 probe-lock 精确钉到当前站点 / 账户 / API Key / 模型
- 如果本地路由代理已经观察到明确终止错误，例如无效 API Key、路径全部暂停、上游 4xx/5xx 或协议不匹配，测试进程会提前停止并显示代理返回的错误摘要
- 如果 CLI 没有向本地路由代理发起请求，结果会提示检查 CLI 登录状态、环境变量代理配置和本机 CLI 可执行性

### 配置文件损坏了怎么办？

- 应用会先校验配置结构
- 若主配置不可读，会尝试从最近备份恢复
- 同时会保留一份损坏副本，便于后续排查

### 如何迁移到新电脑？

- 通过导出/导入配置迁移
- 或直接恢复本地备份
- 或配置 WebDAV 后从云端恢复

### 支持哪些系统？

- Windows
- macOS
- Linux

其中 Linux 打包已提供，但你的实际兼容性仍取决于本地 Chromium 环境和桌面依赖。
