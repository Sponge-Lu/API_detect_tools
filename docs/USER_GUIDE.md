# API Hub Management Tools - 用户使用指南

## 简介

**API Hub Management Tools** 用于管理和运维多个 API 中转站，适合同时维护多站点、多账号、CLI 配置和代理路由的用户。

当前版本为 **v3.0.1**。相较 `v2.1.24`，最明显的变化是：

- 新增 Route 工作台
- 支持账号级自动刷新
- 主界面重构为侧边栏工作台
- 主题系统收敛为 `Light` / `Dark`

## 核心功能

### 1. 站点与账号管理

- 在一个页面中维护多个 API 站点
- 支持站点分组、排序、搜索和缓存数据展示
- 支持一个站点绑定多个账号
- 支持按账号配置自动刷新开关和间隔
- 支持按浏览器 Profile 直接打开站点

### 2. 自动认证与登录

- 添加站点时可直接输入 URL，通过浏览器登录后自动捕获凭证
- 若登录后站点未返回 `access_token`，系统会尝试自动补建令牌
- 遇到 Cloudflare 或挑战页时，会回退到浏览器模式
- 登录浏览器与检测浏览器已分离，减少批量检测和登录流程冲突

### 3. 站点检测与统计

- 查看余额、今日消费、请求数、RPM、TPM、模型数量等指标
- 批量检测时支持逐站即时更新
- 支持站点级与账号级自动刷新
- 支持全局排序，排序状态会自动保存

### 4. API Key 与签到

- 可视化创建、删除、查看 API Key
- 支持配额、过期时间、分组等参数
- 自动检测签到能力
- 支持单站签到与批量签到
- 支持浏览器模式完成被挑战站点的签到

### 5. CLI 配置与兼容性

支持以下 CLI：

- Claude Code
- Codex
- Gemini CLI

当前行为说明：

- **Codex**：只测试并生成 Responses API 配置，不再使用 Chat Completions 模式
- **Gemini CLI**：保留 Native / Proxy 双端点探测，实际支持判断以 Native 为主

你可以完成以下操作：

- 为每个 CLI 选择 API Key 和模型
- 查看配置预览
- 直接编辑配置内容
- 一键写入本地配置文件
- 选择合并模式或覆盖模式应用配置
- 管理独立于站点检测结果的“自定义 CLI 配置”

### 6. Route 工作台

这是 `v3.0.1` 新增的工作区，分为三部分：

#### 模型重定向

- 配置统一服务端口、统一 API Key
- 创建或修改路由规则
- 维护模型注册表与覆盖项

#### CLI 可用性

- 查看 CLI 探测最新结果
- 手动触发探测
- 查看历史记录与诊断信息

#### 代理统计

- 查看代理运行状态
- 查看健康检查结果
- 查看聚合统计与分布数据

### 7. Linux Do Credit

- 读取 `credit.linux.do` 的余额、收入、支出、交易记录
- 查看最近统计数据
- 选择支持 LDC 支付的站点并发起充值跳转

### 8. 备份、恢复与更新

- 本地自动备份配置
- 从历史备份恢复
- 使用 WebDAV 进行云端备份
- 应用内检查更新、下载和安装
- 配置损坏时会优先尝试从最近备份恢复

## 常用使用路径

### 添加站点

1. 打开 `站点管理`
2. 新建站点并填写 URL
3. 选择自动登录或手动输入凭证
4. 登录完成后等待系统抓取账号信息

### 配置账号级自动刷新

1. 打开站点详情或账户相关入口
2. 为目标账号设置 `auto_refresh`
3. 设置刷新间隔，最小为 15 分钟
4. 保存后等待定时器生效

### 生成 Codex 配置

1. 打开 `自定义 CLI` 或站点 CLI 配置入口
2. 选择 `Codex`
3. 选择 API Key 和模型
4. 预览生成结果
5. 点击应用

说明：当前生成的 Codex 配置固定使用 `Responses API`。

### 使用 Route 工作台

1. 在侧边栏进入 `模型重定向`、`CLI 可用性` 或 `代理统计`
2. 根据页面类型维护规则、探测配置或查看统计
3. 修改内容会写回 `config.routing`

## FAQ

### 为什么登录浏览器没有立刻关闭？

- 可能仍在等待你完成登录或通过挑战页
- 当前版本会把登录流程放在独立登录浏览器中，避免打断检测任务
- 若长时间无进展，请检查是否仍停留在验证页或登录页

### 为什么有些站点需要浏览器模式才能检测？

- 某些站点接口会被 Cloudflare、WAF 或登录态限制拦截
- 当前版本会统一识别挑战页并切换到浏览器上下文继续获取数据

### 为什么 Codex 不再测试 Chat Completions？

- 当前版本已经把 Codex 配置与测试收敛到 Responses API
- 这是 `v3.0.1` 相比 `v2.1.24` 的一个行为变化

### 配置文件损坏了怎么办？

- 应用会先校验配置结构
- 若主配置不可读，会尝试从最近备份恢复
- 同时会保留一份损坏副本，便于后续排查

### 如何迁移到新电脑？

- 通过导出/导入配置迁移
- 或直接恢复本地备份
- 或配置 WebDAV 后从云端恢复

### 支持哪些系统？

- Windows
- macOS
- Linux

其中 Linux 打包已提供，但你的实际兼容性仍取决于本地 Chromium 环境和桌面依赖。
